## Generated Resource Pattern

The ACI specialist template includes:

- Namespace: `openclaw-aci`
- ServiceAccount: `openclaw-aci`
- Role: namespace-scoped read-only access to `configmaps`, `events`, and `pods`
- RoleBinding: binds only the namespace-local ServiceAccount
- OpenClawInstance: PVC-backed workspace with `openai-api` provided through `envFrom`
- CiliumNetworkPolicy: DNS, OpenAI, and GitHub egress allowed; metadata endpoints denied

## Required Human Review Before New Specialist Promotion

For every new specialist generated by `openclaw-factory`, require:

- Namespace name and labels are correct.
- RBAC is namespace-scoped and least-privilege.
- Cilium policy allows only required destinations.
- LLM/API secrets are referenced, not embedded.
- Workspace memory files match the specialist's role.
- Hubble evidence checklist is included.
